# 2. Risk Checklist

- **Scope Creep:** Risk of adding extra features outside simple todo core (prevent with strict task list)
- **Quality Skips:** Potential to skip QA or pass with insufficient evidence (mitigate with enforced evidence requirement)
- **Miscommunication:** Ambiguous handoffs causing developer to implement non-required features (avoid with concrete instructions referencing spec)
- **Technical Ambiguity:** Unclear architecture decisions (simple local storage vs backend) can delay development
- **Agent Failures:** Specialist agent spawn failure or stalls (implement retry logic)
- **UI/UX Usability:** Overcomplicating UI rather than keeping minimal (controlled by UX foundation phase)
- **Testing Coverage:** Missing critical functionality in QA tests (EvidenceQA instructed to verify all CRUD flows)
